Hugging Face API Token Guide

This guide will help you create and configure a Hugging Face API token with the necessary permissions for use with the MCP Configuration Tool.

Note: A Hugging Face account is required to create an API token. If you don't have an account, you'll need to create one first.

Creating a Hugging Face API Token

1 Sign in to Hugging Face

Go to the Hugging Face login page and sign in to your account. If you don't have an account, you'll need to create one.

2 Navigate to Settings

Once signed in, click on your profile picture in the top-right corner of the page and select "Settings" from the dropdown menu.

3 Access the API Tokens Section

In the left sidebar of the Settings page, click on "Access Tokens".

4 Create a New Token

Click the "New token" button to create a new API token.

5 Configure Token Permissions

Fill in the following information:

  • Token name: Provide a meaningful name (e.g., "MCP Config Tool")
  • Role: Select "Read" for basic access, or "Write" if you plan to use features that require write access

For most MCP configurations, selecting the "Read" role is sufficient.

6 Generate the Token

Click the "Generate a token" button to create your token.

Important: Copy your token immediately and store it in a safe place. You will not be able to view it again after leaving this page.

7 Use the Token in MCP Configuration Tool

Paste the token into the Hugging Face token field in the MCP Configuration Wizard.

Scope and Permissions

Different MCP server configurations may require different permission levels:

  • Read access: Allows querying models, embeddings, and other read-only operations
  • Write access: Necessary for operations that modify resources, such as fine-tuning models
Recommendation: Start with the minimum required permissions (Read) and upgrade only if needed for specific operations.

Token Security Best Practices

  • Never share your API tokens publicly
  • Avoid committing tokens to version control systems
  • Revoke tokens that are no longer in use
  • Create different tokens for different applications
Security Warning: Anyone with your token can make API calls on your behalf, potentially incurring usage charges or accessing private information.

Troubleshooting Token Issues

If you encounter issues with your Hugging Face token:

  1. Verify that your token has not expired
  2. Check that your token has the necessary permissions
  3. Regenerate your token if you suspect it has been compromised
  4. Ensure you're correctly copying the full token string without extra spaces
